## 三、关键知识点

### 1. GitHub Pages 站点数量
- **用户站点**：1个（username.github.io）
- **项目站点**：无限个

### 2. 自定义域名配置
- **DNS CNAME 记录值**：统一填 `用户名.github.io`
- **仓库 CNAME 文件**：填写自定义域名
- **不要**在 DNS 记录值中添加仓库路径

### 3. CNAME 文件作用
- 告诉 GitHub 请求的域名属于哪个仓库
- 必须放在仓库根目录
- 内容格式：`your-domain.com`（无 https://）

### 4. 部署后更新
- 推送代码到 main 分支
- GitHub Actions 自动部署
- 等待 1-2 分钟生效

## 四、常见问题

### Q: 自定义域名配置后无法访问？
A: 检查 DNS 记录是否正确，CNAME 值应为 `用户名.github.io`（不含仓库路径）

### Q: GitHub Pages 站点数量限制？
A: 用户站点 1 个，项目站点无限

### Q: 如何管理多个站点？
A: 为每个仓库绑定不同子域名

## 六、常用 GitHub API

```bash
# 查看 GitHub Pages 状态
curl -H "Authorization: token TOKEN" \
  https://api.github.com/repos/用户名/仓库名/pages

# 启用 GitHub Pages
curl -X POST -H "Authorization: token TOKEN" \
  -d '{"build_type":"workflow","source":{"branch":"main","path":"/"}}' \
  https://api.github.com/repos/用户名/仓库名/pages

# 禁用 GitHub Pages
curl -X DELETE -H "Authorization: token TOKEN" \
  https://api.github.com/repos/用户名/仓库名/pages
```
